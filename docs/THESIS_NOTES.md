# 论文写作笔记

本文题目为“基于链上数据的加密货币收益预测模型研究”。写作重点不应停留在“哪个模型指标最高”，而应围绕链上数据是否具有增量预测价值、预测信号能否转化为交易收益，以及不同模型和不同任务之间是否存在稳定差异展开。

## 1. 核心研究问题

论文主线可以组织为三个问题：

1. 链上数据相较于价格与技术面信息，是否能够提供额外预测信息。
2. 分类任务和回归任务在预测表现与交易转化能力上是否存在差异。
3. 预测指标较好的模型，是否能够稳定转化为更好的回测收益。

这三个问题分别对应第 5 章预测结果分析和第 6 章经济价值分析。第 3、4 章主要负责说明数据、特征、模型和验证框架，为后续结果解释提供方法基础。

## 2. 章节写作对应关系

| 章节 | 写作重点 | 项目依据 |
| --- | --- | --- |
| 第 3 章 | 数据来源、特征工程、特征筛选、防泄漏和 walk-forward 设计 | `configs/`、`src/ingest/`、`src/features/`、`src/datasets/` |
| 第 4 章 | 分类/回归任务、机器学习/深度学习模型、参数设置和评价指标 | `src/models/`、`src/evaluation/`、`src/cli.py` |
| 第 5 章 | 分类预测结果、回归预测结果和数据集变体比较 | `data/features/*_metrics.json`、`reports/summary/` |
| 第 6 章 | 交易信号映射、回测表现、激活策略和稳健性分析 | `data/features/*_backtest_sensitivity.csv`、`reports/experiments/` |
| 第 7 章 | 主要结论、研究不足和未来改进方向 | 第 5、6 章归纳 |

## 3. 结果表格优先级

正文结果应优先用表格支撑，再用图形辅助解释。建议优先整理以下表格：

| 表格 | 用途 |
| --- | --- |
| 分类任务预测结果表 | 展示 Accuracy、Precision、Recall、F1 |
| 回归任务预测结果表 | 展示 RMSE、MAE、IC、RankIC |
| 数据集变体比较表 | 比较 `onchain`、`ta`、`all` 和筛选后特征集 |
| 原始策略回测表 | 展示基础交易映射下的收益表现 |
| 激活策略回测表 | 比较全程交易、阈值激活和分位数策略 |
| 分阶段稳健性表 | 检查不同减半周期下结果是否稳定 |

所有正文表格应尽量使用三线表格式，并避免一次性塞入过多模型。正文中可展示代表性结果，完整结果保留在附录或结果文件中。

## 4. 图形建议

图形应服务于解释，不应替代表格。建议优先保留：

- 研究流程图：展示数据、特征、模型、预测和回测的整体流程。
- Walk-forward 划分示意图：说明样本外验证与防泄漏逻辑。
- 分类任务 F1 对比图：辅助展示分类任务表现差异。
- 回归任务 IC 或 RMSE 对比图：辅助展示回归任务表现差异。
- 策略净值曲线：展示代表模型的交易结果。
- 回撤曲线：展示收益风险特征。

如果篇幅有限，优先保留研究流程图、Walk-forward 图、代表性净值曲线和回撤曲线。

## 5. 结果解释口径

写作时应避免将单一指标解释为绝对最优。较稳妥的表达方式是：

- 分类任务中，F1 更适合作为综合方向预测指标，但仍需结合 Precision 和 Recall 判断模型是否偏向单一类别。
- 回归任务中，RMSE 和 MAE 衡量误差大小，IC 和 RankIC 更适合判断收益预测信号的相关性和排序能力。
- 交易回测中，累计收益和年化收益不能单独说明策略优劣，需要结合 Sharpe 比率、最大回撤和换手率。
- 预测表现较好不必然意味着交易收益更好，因此第 5 章和第 6 章需要分开讨论。

## 6. 需要主动说明的局限

论文中应主动讨论以下限制：

- 免费链上数据覆盖有限，无法使用部分高级链上指标。
- 比特币市场具有强非平稳性，模型表现可能受市场阶段影响。
- 深度学习模型对时间窗口、初始化和训练路径较敏感。
- 回测仍存在简化假设，例如交易成本、滑点和做空约束没有完全贴近真实交易环境。
- 特征筛选能够降低冗余，但不能完全消除过拟合风险。

这些内容适合放在第 7 章“研究不足与展望”中，也可以在第 6 章稳健性分析后简要呼应。

## 7. 写作风格提示

正文应尽量使用“本研究”或“系统”作为主语，减少反复使用“本文”。方法章节应突出可比较、可复现和防泄漏；结果章节应突出数据依据，而不是先给结论再找解释。

较推荐的表达方式：

```text
在统一样本外验证框架下，不同模型的表现差异主要来自模型结构和输入特征，而非样本切分或评价口径差异。
```

不推荐的表达方式：

```text
该模型效果最好，因此说明链上数据一定有效。
```

更稳妥的写法是：

```text
该结果表明，在当前样本区间和验证框架下，筛选后的链上特征在部分模型中表现出一定增量价值，但其稳定性仍需结合回测结果和分阶段分析进一步判断。
```
